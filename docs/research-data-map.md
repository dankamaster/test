# Research Data Map

This document explains how research inputs were transformed into the final page.

## Input groups

### SERP research

Collected:

- recurring domains;
- recurring content formats;
- attorney-facing resource patterns;
- nonprofit/legal-aid resources;
- country conditions vendors.

Used for:

- article structure;
- FAQ topics;
- CTA design;
- trust architecture.

## PDF corpus

Collected:

- COI methodology documents;
- legal-aid guides;
- asylum documentation references;
- country conditions examples.

Used for:

- terminology;
- source hierarchy;
- evidence workflow.

## Legal source layer

Collected:

- USCIS resources;
- EOIR / DOJ resources;
- UNHCR resources;
- asylum support resources.

Used for:

- legal-safety boundaries;
- attorney-review positioning;
- YMYL controls.

## Keyword layer

Collected:

- keyword planner exports;
- 1-gram terms;
- 2-gram terms;
- 3-gram terms;
- intent groupings.

Used for:

- title;
- H2 structure;
- FAQ;
- semantic coverage.

## Engineering layer

Used for:

- PWA setup;
- schema;
- accessibility;
- privacy layer;
- QA automation.

## Final transformation

SERP → Sources → PDFs → Legal review → Keyword mapping → Information architecture → Article → Conversion layer → QA automation
