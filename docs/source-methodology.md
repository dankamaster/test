# Source Validation Methodology

## Goal

Create a transparent, repeatable process for validating sources used in country conditions research, COI reports, and attorney-facing documentation resources.

## Source classes

### Class A — Primary authority

Examples:

- USCIS
- EOIR / DOJ
- US Code
- eCFR
- UNHCR

Use:

- legal framework;
- procedure;
- official guidance;
- terminology.

### Class B — COI methodology and asylum research

Examples:

- EUAA
- CGRS
- Refworld

Use:

- source evaluation;
- country information methodology;
- documentation structure.

### Class C — Practical legal-aid resources

Examples:

- Tahirih
- Immigration Equality
- Immigration Justice Campaign
- NIJC

Use:

- workflow examples;
- practical documentation guidance;
- sample structures.

### Class D — Commercial competitors

Examples:

- CountryConditions.expert

Use:

- positioning analysis;
- conversion analysis;
- market expectations.

Not used as authority for legal claims.

## Validation fields

Each source record should include:

- source name;
- URL;
- jurisdiction;
- publication/update date;
- access date;
- reliability class;
- purpose;
- supported claim(s);
- notes.

## Citation principle

Visible article claims should ultimately be traceable to identifiable sources. Generic statements such as "sources include X" are insufficient for production-grade YMYL content.
