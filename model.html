AI Working Model Used for This SEO/PWA Assignment

This page documents the structured AI working model used to research, plan, build, review, and harden the COI report SEO/PWA test assignment.

The model was used to accelerate research, structure thinking, create QA gates, and surface risks. It does not replace legal review, source validation, or human editorial responsibility.

Core Prompt Frame

The working instruction was to operate as a senior SEO, performance, technical SEO, YMYL content, legal-safety, conversion, and QA system. The goal was not only to write a page, but to build a traceable, source-aware, testable assignment package.

Goal: Build a standalone SEO/PWA page for a legal-sensitive COI/asylum topic.
Audience: immigration attorneys, paralegals, law firms, and legal teams.
Constraints: no legal advice, no invented claims, attorney-controlled workflow, privacy-safe email conversion, no tracking by default.
Quality bar: source-backed, technically inspectable, mobile-first, schema-aware, conversion-aware, and reviewable by Codex/Claude.

Self-Diagnostic Gates

  • SEO gate: primary keyword, secondary clusters, title, H1/H2, FAQ, schema, internal links.
  • Legal/YMYL gate: no legal advice, no guaranteed outcome, counsel controls strategy.
  • Source gate: distinguish official authority, methodology, legal-aid resources, PDFs, and competitors.
  • Privacy gate: do not request sensitive personal identifiers; no analytics before consent.
  • Conversion gate: email-first CTA, legal-team language, request scope without over-collecting facts.
  • Engineering gate: one H1, schema validity, accessibility smoke tests, repo structure, PWA basics.
  • Red-team gate: external Codex review and hardening plan.

Known Model Limitations

  • The model accelerates structure and QA, but final legal review still requires qualified counsel.
  • Source inventories need production URLs, access dates, and claim-level citations before final publication.
  • Lighthouse scores must be measured after deployment, not assumed.
  • Production EEAT requires real author/reviewer identities, not only an Editorial Team placeholder.
